ในทั้งสองกรณี shob จะทำงานบน GitLab runners ของคุณ

---

## GitLab CI

shob ทำงานในไปป์ไลน์ GitLab ปกติ คุณสามารถสร้างมันลงในไปป์ไลน์เป็น [องค์ประกอบ CI](https://docs.gitlab.com/ee/ci/components/)

ที่นี่เรากำลังใช้คอมโพเนนต์ CI/CD ที่สร้างโดยชุมชนสำหรับ shob — [nagyv/gitlab-shob](https://gitlab.com/nagyv/gitlab-shob)

---

### คุณสมบัติ

- **ใช้การกำหนดค่าแบบกำหนดเองต่องาน**: กำหนดค่า shob ด้วยไดเร็กทอรีการกำหนดค่าแบบกำหนดเอง เช่น `./config/#custom-directory` เพื่อเปิดหรือปิดฟังก์ชันการทำงานต่อการเรียกใช้ shob
- **การตั้งค่าขั้นต่ำ**: องค์ประกอบ CI จะตั้งค่า shob ในเบื้องหลัง คุณเพียงแค่ต้องสร้างการกำหนดค่า shob และข้อความแจ้งเริ่มต้นเท่านั้น
- **ยืดหยุ่น**: ส่วนประกอบ CI รองรับอินพุตหลายอินพุตเพื่อปรับแต่งลักษณะการทำงาน

---

### ตั้งค่า

1. จัดเก็บการรับรองความถูกต้อง shob ของคุณ JSON เป็นประเภทไฟล์ CI ตัวแปรสภาพแวดล้อมภายใต้ **การตั้งค่า** > **CI/CD** > **ตัวแปร** ตรวจสอบให้แน่ใจว่าได้ทำเครื่องหมายว่าเป็น "ปิดบังและซ่อนไว้"
2. เพิ่มสิ่งต่อไปนี้ในไฟล์ `.gitlab-ci.yml` ของคุณ

   ```yaml title=".gitlab-ci.yml"
   include:
     - component: $CI_SERVER_FQDN/nagyv/gitlab-shob/shob@2
       inputs:
         config_dir: ${CI_PROJECT_DIR}/shob-config
         auth_json: $shob_AUTH_JSON # The variable name for your shob authentication JSON
         command: optional-custom-command
         message: "Your prompt here"
   ```

หากต้องการข้อมูลเพิ่มเติมและกรณีการใช้งาน [ตรวจสอบเอกสาร](https://gitlab.com/explore/catalog/nagyv/gitlab-shob) สำหรับส่วนประกอบนี้

## GitLab Duo

shob ทำงานร่วมกับเวิร์กโฟลว์ GitLab ของคุณ
พูดถึง `@shob` ในความคิดเห็น แล้ว shob จะดำเนินการงานภายในไปป์ไลน์ GitLab CI ของคุณ

---

### คุณสมบัติ

- **Issue Triage**: ขอให้ shob ตรวจสอบปัญหาและอธิบายให้คุณทราบ
- **แก้ไขและนำไปใช้**: ขอให้ shob แก้ไขปัญหาหรือใช้ฟีเจอร์
  มันจะสร้างสาขาใหม่และขอรวมการเปลี่ยนแปลง
- **ปลอดภัย**: shob ทำงานบนนักวิ่ง GitLab ของคุณ

---

### ตั้งค่า

shob ทำงานในไปป์ไลน์ GitLab CI/CD ของคุณ นี่คือสิ่งที่คุณจะต้องตั้งค่า:

:::tip
ตรวจสอบ [**เอกสาร GitLab**](https://docs.gitlab.com/user/duo_agent_platform/agent_assistant/) เพื่อดูคำแนะนำล่าสุด
:::

1.  กำหนดค่าสภาพแวดล้อม GitLab ของคุณ
2.  ตั้งค่า CI/CD
3.  รับผู้ให้บริการโมเดล AI API คีย์
4.  สร้างบัญชีบริการ
5.  กำหนดค่าตัวแปร CI/CD
6.  สร้างไฟล์กำหนดค่าโฟลว์ ต่อไปนี้เป็นตัวอย่าง:

        <details>

    <summary>Flow configuration</summary>

    ```yaml
    image: node:22-slim
    commands:
      - echo "Installing shob"
      - npm install --global shob-ai
      - echo "Installing glab"
      - export GITLAB_TOKEN=$GITLAB_TOKEN_shob
      - apt-get update --quiet && apt-get install --yes curl wget gpg git && rm --recursive --force /var/lib/apt/lists/*
      - curl --silent --show-error --location "https://raw.githubusercontent.com/upciti/wakemeops/main/assets/install_repository" | bash
      - apt-get install --yes glab
      - echo "Configuring glab"
      - echo $GITLAB_HOST
      - echo "Creating shob auth configuration"
      - mkdir --parents ~/.local/share/shob
      - |
        cat > ~/.local/share/shob/auth.json << EOF
        {
          "anthropic": {
            "type": "api",
            "key": "$ANTHROPIC_API_KEY"
          }
        }
        EOF
      - echo "Configuring git"
      - git config --global user.email "shob@gitlab.com"
      - git config --global user.name "shob"
      - echo "Testing glab"
      - glab issue list
      - echo "Running shob"
      - |
        shob run "
        You are an AI assistant helping with GitLab operations.

        Context: $AI_FLOW_CONTEXT
        Task: $AI_FLOW_INPUT
        Event: $AI_FLOW_EVENT

        Please execute the requested task using the available GitLab tools.
        Be thorough in your analysis and provide clear explanations.

        <important>
        Please use the glab CLI to access data from GitLab. The glab CLI has already been authenticated. You can run the corresponding commands.

        If you are asked to summarize an MR or issue or asked to provide more information then please post back a note to the MR/Issue so that the user can see it.
        You don't need to commit or push up changes, those will be done automatically based on the file changes you make.
        </important>
        "
      - git checkout --branch $CI_WORKLOAD_REF origin/$CI_WORKLOAD_REF
      - echo "Checking for git changes and pushing if any exist"
      - |
        if ! git diff --quiet || ! git diff --cached --quiet || [ --not --zero "$(git ls-files --others --exclude-standard)" ]; then
          echo "Git changes detected, adding and pushing..."
          git add .
          if git diff --cached --quiet; then
            echo "No staged changes to commit"
          else
            echo "Committing changes to branch: $CI_WORKLOAD_REF"
            git commit --message "Codex changes"
            echo "Pushing changes up to $CI_WORKLOAD_REF"
            git push https://gitlab-ci-token:$GITLAB_TOKEN@$GITLAB_HOST/gl-demo-ultimate-dev-ai-epic-17570/test-java-project.git $CI_WORKLOAD_REF
            echo "Changes successfully pushed"
          fi
        else
          echo "No git changes detected, skipping push"
        fi
    variables:
      - ANTHROPIC_API_KEY
      - GITLAB_TOKEN_shob
      - GITLAB_HOST
    ```

        </details>

คุณสามารถดูคำแนะนำโดยละเอียดได้ที่ [GitLab CLI agent docs](https://docs.gitlab.com/user/duo_agent_platform/agent_assistant/)

### ตัวอย่าง

นี่คือตัวอย่างบางส่วนของวิธีที่คุณสามารถใช้ shob ใน GitLab

:::tip
คุณสามารถกำหนดค่าให้ใช้วลีทริกเกอร์ที่แตกต่างจาก `@shob`
:::

- **อธิบายปัญหา**

  เพิ่มความคิดเห็นนี้ในปัญหา GitLab

  ```
  @shob explain this issue
  ```

  shob จะอ่านปัญหาและตอบกลับพร้อมคำอธิบายที่ชัดเจน

- **แก้ไขปัญหา**

  ในปัญหา GitLab ให้พูดว่า:

  ```
  @shob fix this
  ```

  shob จะสร้างสาขาใหม่ ดำเนินการเปลี่ยนแปลง และเปิดคำขอรวมที่มีการเปลี่ยนแปลง

- **ตรวจสอบคำขอรวม**

  แสดงความคิดเห็นต่อไปนี้ในคำขอรวม GitLab

  ```
  @shob review this merge request
  ```

  shob จะตรวจสอบคำขอรวมและให้ข้อเสนอแนะ
